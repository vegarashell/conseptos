# conseptos
#1. ¿Qué es una Base De Datos Relacional?
Una Base de Datos Relacional (RDB) es un tipo de base de datos que organiza los datos en tablas, que están relacionadas entre sí mediante claves primarias y claves foráneas. Utiliza el modelo relacional para almacenar, organizar y manipular datos de manera estructurada y eficiente. Las bases de datos relacionales utilizan un lenguaje estructurado de consulta llamado SQL (Structured Query Language) para gestionar y realizar consultas sobre los datos.
#2. ¿Qué es una Tabla en el contexto Base De #Datos Relacionales?
En el contexto de las bases de datos relacionales, una tabla es una estructura que organiza los datos en filas y columnas. Cada columna representa un atributo o campo de los datos, mientras que cada fila (o renglón) representa un registro o entrada individual. Las tablas se utilizan para almacenar y organizar datos de manera estructurada.
#3. ¿Qué es una relación en el contexto de #Base De Datos Relacionales?
Una relación en una base de datos relacional se refiere a la asociación entre diferentes tablas mediante claves. Las relaciones se establecen utilizando claves primarias (identificadores únicos en una tabla) y claves foráneas (campos en una tabla que apuntan a la clave primaria de otra tabla). Estas relaciones permiten combinar datos de diferentes tablas en consultas complejas.
#4. ¿Qué representa una columna en el contexto #de Base De Datos Relacionales?
Una columna en una tabla de base de datos relacional representa un atributo o campo específico que describe una característica de los datos. Cada columna tiene un tipo de dato asociado (por ejemplo, texto, número, fecha) y puede contener valores para todos los registros de la tabla.
#5. ¿Qué representa un renglón en el contexto #de Base De Datos Relacionales?
Un renglón (o fila) en una tabla representa un registro o una entrada individual que contiene datos para todos los atributos definidos por las columnas de la tabla. Cada renglón debe ser único y generalmente se identifica por una clave primaria.
#6. ¿Qué es un motor de base de datos #relacional?
Un motor de base de datos relacional es el software o sistema que gestiona la base de datos, permitiendo la creación, modificación, consulta y administración de bases de datos relacionales. El motor se encarga de almacenar los datos, garantizar la integridad de la información, y ejecutar consultas SQL.

#8. ¿Cuál es el motor de base datos ligero que no necesita un servidor?
SQLite es un motor de base de datos ligero que no necesita un servidor. Es una base de datos embebida, lo que significa que se integra directamente en la aplicación sin requerir un servidor separado para gestionar la base de datos.
#9. ¿Qué significa SQL en el contexto de base #datos relacionales?
SQL significa Structured Query Language (Lenguaje de Consulta Estructurado) y es el lenguaje estándar utilizado para gestionar bases de datos relacionales. SQL permite crear, modificar, eliminar y consultar datos en una base de datos.
#10. ¿Cuáles son los elementos de un sistema #de información basado en base de datos #relacionales?
Los elementos principales de un sistema de información basado en base de datos relacionales son:

Datos: Información almacenada en las tablas.

Base de datos: Colección organizada de datos relacionados.

Motor de base de datos: El software que gestiona la base de datos.

Usuarios: Personas o aplicaciones que interactúan con la base de datos.

Lenguaje de consulta (SQL): El lenguaje utilizado para interactuar con los datos.

Interfaz de usuario o aplicaciones: Herramientas o interfaces que permiten a los usuarios interactuar con la base de datos.
